Configuration Management Plan Il Project Configuration Management (PCM) è un processo gestionale finalizzato a stabilire e mantenere la coerenza tra le performance dei deliverable (prodotto, processo, documento) e dei loro attriubuti con i requisiti di progetto, mantenendoli allineati per tutto il ciclo di vita del progetto.
Il Configuration Management Plan (CMP) documenta la modalità con cui la configurazione sarà gestita. Elenca tutti gli item che fanno parte della configurazione e quali di essi sono soggetti a formale controllo delle modifiche.
Per un corretto approccio alla gestione della configurazione i seguenti elementi devono essere considerati:
  • Configuration Management Planning - introduce il concetti del PCM e le loro relazioni nel project management;
  • Configuration Identification - identifica gli item di progetto sottoposti al controllo di configurazione.
  • Configuration Change Management - fornisce le linee guida per l'identificazione delle change nel progetto.
  • Configuration Status Accounting and Metrics - descrive strumenti e tecniche che possono essere utilizzate per misurare i progressi e la maturità degli item sottoposti a configurazione.
  • Configuration Verification and Audits - descrive come verificatori indipendenti possano qiutare il project team nel confermare ch il lavoro fatto sia esattamente quello inteso.
