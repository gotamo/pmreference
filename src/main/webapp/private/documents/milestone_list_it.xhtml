Milestone List Una milestone rappresenta un momento nel tempo in cui i membri del team hanno raggiunto un evento importante per il progetto. La sua caratteristica è quella di avere zero durata. Le milestone sono fondamentali nella gestione dei progetti in quanto aggiungono valore alla pianificazione del progetto e permettono ai project manager di determinare se i progetto è in ritardo o meno. In poche parole, sono utilizzate per monitorare lo stato di avanzamento del progetto.
Questo è il motivo per cui i project manager devono creare la Milestone List, ovvero la lista di tutte le milestone che caratterizzano il progetto. Tale documento indica anche se una milestone particolare è critica, se è richiesta dal contratto o è opzionale per il progetto.
È responsabilità del project manager creare la lista dele milestone che spesso è ottenuta per analogia utilizzando informazioni storiche di progetti simili. Trattandosi di un documento di progetto la sua realizzazione richiede input provenienti dagli altri membri del team di progetto. Esempi di input per questo particolare documento includono:
  • Scope Baseline;
  • Schedule Management Plan;
  • Organizational Process Assets (OPA);
  • Enterprise Environmental Factors (EEF).
